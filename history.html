Typing a key bound to backward-delete-char while in input_search_mode will just erase the last character of the search instead of terminating input_search_mode. C-r/C-s will look for the next match in the same direction, or will switch directions. If the current search string is empty when C-r/C-s would cause a search for the next match, the last search is recalled. Typing C-g will quit searching and return everything to the state it was in before searching started. C-l will refresh the screen without exiting. Any other special character exits input_search_mode, as was true before, but now the history position is set based on where search stopped. Can now search for the string "0"; could not before. Case folded searching works, configurable with $config{case_fold_search}. [DCL]
